認識指標Pointers · Definition · Concept

1. 定義（Definition）

指標（Pointer）是一種特殊變數——它不儲存實際資料值，而是儲存記憶體位址。 透過指標我們可以「間接」存取另一個變數的內容，這種機制稱為間接參考（Indirection）。

白話版

指標就像「房間門口的指示牌」——牌子上寫的是房間號碼（位址），而不是房間裡的東西（資料值）。

2. 為什麼需要指標？

  • 能高效處理連續記憶體（陣列）
  • 能實現傳址呼叫（修改原始變數）
  • 是許多進階資料結構（如 Linked List、Tree、Graph）的基礎
  • 動態記憶體配置（new / malloc）的必備工具

3. C++ vs Python 的觀念對應

概念對照

C++ 有明確的指標語法（*、&）；Python 所有變數本質上都是物件參考（Reference），對可變物件（list、dict）的行為類似「傳址」。

4. 圖解：指標與記憶體的基本關係

記憶體位址 vs 變數視角

5. 互動範例 — 指標基本操作

C++
int num1 = 10; int *address1 = &num1; // 宣告指標並取址 cout << *address1; // 輸出 10（取值） *address1 = 20; // 修改 num1 → 20
Python（無指標但有參考）
num1 = 10 print(id(num1)) # 顯示記憶體位址 lst = [10, 20, 30] print(id(lst)) # list 可變，類似指標
APCS 高頻陷阱

① 宣告 int *x, y; → x 是指標，y 是一般整數（不是兩個指標！）
② 未初始化指標（野指標） → 可能指向垃圾記憶體，造成程式崩潰
③ 直接給指標數值 int *p = 10; → 不合法，必須給「位址」

多重指標Pointer to Pointer · Multi-level Indirection

1. 定義

雙重指標：int **ptr2;——指向另一個指標的指標。 每多一個 *，就多一層間接。

2. 圖解：雙重指標的記憶體鏈接

3. C++ vs Python 對照

C++
int a = 10; int *ptr1 = &a; // ptr1 指向 a int **ptr2 = &ptr1; // ptr2 指向 ptr1 cout << **ptr2; // 輸出 10
Python（用 list 模擬）

指標運算Pointer Arithmetic

1. 核心原理

指標可進行 +、++、-、-- 運算，但移動單位是「資料型態的大小」，而非 1 byte。

關鍵觀念

指標 + 1 不是加 1 byte，而是移動一個元素的記憶體空間（int 通常移動 4 bytes，double 移動 8 bytes）。

指標與陣列Arrays as Pointer Constants

1. 核心定義

陣列名稱本身就是指標常數，指向第一個元素。所以：

⭐ 必背等價式

arr[i] ≡ *(arr + i) ≡ *(i + arr) ≡ i[arr]

2. 範例對照

C++
int arr[5] = {10, 20, 30, 40, 50}; int *ptr = arr; // arr 即位址 cout << *ptr; // 10 cout << *(ptr + 2); // 30 cout << ptr[3]; // 40
Python
arr = [10, 20, 30, 40, 50] print(arr[0]) # 10 print(arr[2]) # 30 # list 本身就是參考，傳給函式可變

指標與鏈結串列Linked List — APCS 高頻命題

1. Linked List 節點結構

每個節點包含「Data（資料）」與「Next Pointer（指向下一個的指標）」。最後一個節點的 next 指向 NULL（C++）或 None（Python）。

Linked List 結構示意
10
→
20
→
30
→
NULL

2. RemoveNextElement 核心邏輯

要刪除 current 的下一個節點，只需把 current.next 改成「下一個的下一個」。原本被指向的節點就會被「跳過」。

C++（陣列模擬版）
list[current].next = list[list[current].next].next; // current.next 跳過原本指向的，直接接到下下個
Python（節點物件版·更直覺）
current.next = current.next.next

3. 互動視覺化：刪除節點

Linked List · 刪除節點動畫
› current = 0（指向第一個節點）。按「刪除」會移除 current 的下一個節點。
APCS 高頻陷阱

① 忘記檢查 next != -1 → 越界 crash
② 改錯對象：要改的是 list[current].next，不是 current 本身
③ 誤以為改 current 變數有用（其實沒改到串列）

考前速查表30-Second Survival Sheet

本章重點整理

  1. 指標儲存位址，* 用來取值，& 用來取址
  2. 未初始化的指標是野指標，極度危險（APCS 高頻錯誤）
  3. 陣列名稱就是指標常數，arr[i] ≡ *(arr + i)
  4. 每多一個 * 就多一層間接參考
  5. 指標運算的單位是資料型態的大小
  6. 傳址呼叫（指標）能讓函數修改原始變數
🎯 指標核心三招
  • 取址：&變數
  • 取值：*指標
  • 宣告：型態 *名稱
🐍 Python 對應
  • 不可變：int、tuple → 像傳值
  • 可變：list、dict → 像傳址
  • id() 看記憶體位址
  • 巢狀 list 模擬多重指標
⚠️ APCS 五大陷阱
  • int *x, y; → y 不是指標
  • 野指標 segfault
  • int *p = 10; 不合法
  • int* vs int**
  • 大小寫敏感（iVal ≠ ival）
🔢 指標層級口訣
  • 看到 & → 多一層 *
  • 看到 * → 少一層 *
  • a → int
  • &a → int*
  • &&a → int**
🔗 Linked List 刪除
  • 核心：跳過下一個
  • current.next = current.next.next
  • 陣列版：list[current].next = list[list[current].next].next
  • 先檢查 next != -1
🎲 printf 格式速記
  • %d 整數
  • %2.2f 小數兩位
  • 沒空格會連在一起
  • 大小寫絕對不能錯
三句話 KO 本章
* = 開門看裡面；& = 寫下房號；多一層 * = 多一個門牌。
傳址 = 共用記憶體 = 函式內改 = 外面也變。
