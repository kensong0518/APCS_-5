01

認識指標

Pointers — 儲存記憶體位址的特殊變數

📖
白話定義

指標（Pointer）就像「房間門口的指示牌」，牌子上寫的是房間號碼（位址），而不是房間裡的東西（資料值）。指標本身不儲存資料，它儲存的是「另一個變數的位置」。

🧠 核心概念：記憶體與位址

電腦的記憶體（RAM）就像一排有編號的格子。每個格子都有獨一無二的「位址」（address）。當我們宣告一個變數，系統會自動分配一個格子給它。

記憶體示意圖（Memory Layout）

🔑 兩個核心運算子

📍
& — 取址運算子（Address-of）

取得變數的記憶體位址

&num → 傳回 num 的位址，例如 0x1000

🔍
* — 取值運算子（Dereference）

透過指標取出所指向的值

*ptr → 前往 ptr 儲存的位址，取出那裡的值

💻 程式碼範例

C++ 
#include <iostream>
using namespace std;

int main() {
    int num = 42;          // 普通整數變數，值為 42
    int* ptr = &num;      // 宣告指標 ptr，儲存 num 的位址

    cout << "num 的值：" << num << endl;    // 42
    cout << "num 的位址：" << &num << endl; // 例如 0x7ffabcd
    cout << "ptr 儲存的值：" << ptr << endl; // 同 &num（位址）
    cout << "*ptr 取值：" << *ptr << endl;   // 42（間接取得 num 的值）

    *ptr = 99;              // 透過指標修改 num 的值
    cout << "修改後 num：" << num << endl; // 99（num 被改了！）

    return 0;
}
Python 
# Python 沒有指標語法，但用 id() 可以看到記憶體位址概念

num = 42
print(f"num 的值：{num}")          # 42
print(f"num 的位址：{id(num)}")    # 類似 &num，顯示物件 id

# Python 的 list 是可變物件，行為類似「傳址」
lst = [42]               # 用 list 包裝來模擬指標
ref = lst               # ref 參考到同一個 list（像指標）

print(f"lst[0] = {lst[0]}")  # 42
ref[0] = 99              # 透過 ref 修改
print(f"修改後 lst[0] = {lst[0]}")  # 99（lst 也被改了！）

print(f"id(lst) == id(ref): {id(lst) == id(ref)}")  # True
⚠️
APCS 高頻陷阱
  • int *x, y; → x 是指標，y 是普通整數（不是兩個指標！）
  • int *p = 10; → 錯誤！指標必須儲存位址，不能直接給值。應為 int *p = &num;
  • 未初始化的指標（野指標/dangling pointer）→ 指向垃圾記憶體，造成程式崩潰

02

宣告指標變數

Pointer Declaration — 語法、初始化、常見錯誤

📐 語法格式

C++ 語法 
資料型態* 指標名稱;             // 宣告（* 可以貼型態或變數名，效果相同）
資料型態* 指標名稱 = &變數;      // 宣告並初始化
資料型態* 指標名稱 = nullptr;   // 初始化為空指標（安全做法）

🔬 完整範例與逐行解釋

C++ — 完整指標操作 
#include <iostream>
using namespace std;

int main() {
    // ── Step 1: 宣告一般變數 ──
    int num = 10;

    // ── Step 2: 宣告指標並取址 ──
    int* p = &num;   // p 儲存 num 的記憶體位址

    // ── Step 3: 讀值 ──
    cout << num << endl;    // 直接讀 → 輸出 10
    cout << *p << endl;     // 間接讀 → 輸出 10（dereference）

    // ── Step 4: 透過指標修改值 ──
    *p = 20;             // 修改 p 指向的記憶體內容，等同 num = 20
    cout << num << endl;    // 輸出 20（num 被改了！）

    // ── Step 5: 指標本身的位址 ──
    cout << &p << endl;     // p 變數本身的位址（指標也佔記憶體！）
    cout << p << endl;      // p 儲存的值（= num 的位址）

    return 0;
}

🗺️ 記憶體分佈圖

記憶體分佈（執行時示意）
0x7fff1000
10
← int num = 10（4 bytes）
0x7fff1004
0x7fff1000
← int* p = &num（8 bytes on 64-bit）
*p → 前往位址 0x7fff1000 → 取出值 10

⚠️ 常見錯誤 vs 正確寫法

❌ 錯誤寫法✅ 正確寫法說明
int *p = 10; int *p = &num; 指標必須儲存位址，不能給整數值
int *x, *y; → 等同 int *x, y; int *x, *y; 每個指標變數前都要加 *
int *p; *p = 5; int n; int *p = &n; *p = 5; 未初始化就取值 → 野指標崩潰
int *p = nullptr; *p = 5; 先讓 p 指向有效記憶體再取值 對 nullptr 取值 → Segmentation fault

03

多重指標

Pointer to Pointer — 雙重、三重指標與 Python 模擬

🔑
核心概念

每多一個 *，就多一層間接。雙重指標（int**）就是「指向指標的指標」。適合處理動態二維陣列或需要在函數內修改指標本身的場合。

🔗 指標鏈示意

雙重指標記憶體鏈接
ptr2 (int**)
0x3096
位於 #3096
→
ptr1 (int*)
0x2022
位於 #4020
→
var (int)
10
位於 #2022
**ptr2 → 先取 *ptr2 得到 ptr1，再取 *ptr1 得到 10
C++ — 雙重指標 
#include <iostream>
using namespace std;

int main() {
    int a = 10;
    int*  ptr1 = &a;      // ptr1 指向 a
    int** ptr2 = &ptr1;   // ptr2 指向 ptr1（雙重指標）

    // 三種取值方式
    cout << a << endl;       // 直接取值 → 10
    cout << *ptr1 << endl;  // 一層取值 → 10
    cout << **ptr2 << endl; // 兩層取值 → 10

    // 透過雙重指標修改 a
    **ptr2 = 99;
    cout << a << endl;       // 輸出 99

    return 0;
}
Python — 巢狀 list 模擬雙重指標 
# Python 用巢狀 list 模擬多重指標
# 必須用 list（可變物件），不能用整數（不可變）

# 對應 C++: int a = 10;
a = [10]

# 對應 C++: int *ptr1 = &a;
ptr1 = [a]

# 對應 C++: int **ptr2 = &ptr1;
ptr2 = [ptr1]

# 取值（對應 **ptr2）
print(ptr2[0][0][0])   # 10 — 三個 [0] = 兩層指標

# 修改值（對應 **ptr2 = 99）
ptr2[0][0][0] = 99
print(a)                # [99] — a 也跟著變了！
print(ptr1)             # [[99]]
print(ptr2)             # [[[99]]]

指標運算
04

指標運算

Pointer Arithmetic — +, -, ++, -- 的真實意義

⚡
核心概念

指標的 +1 不是加 1 個 byte，而是移動「一個元素大小」的記憶體空間。int 指標 +1 實際移動 4 bytes（sizeof(int) = 4）。

🧮 運算規則

int 指標運算（每格 4 bytes）
C++ — 指標運算完整範例 
#include <iostream>
using namespace std;

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int* ptr = arr;   // ptr 指向 arr[0]（陣列名稱 = 第一個元素的位址）

    cout << *ptr << endl;       // 10（arr[0]）
    cout << *(ptr + 1) << endl; // 20（arr[1]，移動 4 bytes）
    cout << *(ptr + 2) << endl; // 30（arr[2]，移動 8 bytes）

    ptr++;                     // ptr 移到 arr[1]
    cout << *ptr << endl;       // 20

    ptr += 2;                  // ptr 移到 arr[3]
    cout << *ptr << endl;       // 40

    // arr[i] 等價於 *(arr + i)
    cout << arr[3] << endl;    // 40 — 同上
    cout << *(arr+3) << endl; // 40 — 完全等價

    return 0;
}

📋 指標運算規則整理

運算實際移動等價於
ptr + nn × sizeof(type) bytes&arr[n]
ptr++1 × sizeof(type) bytes移到下一個元素
ptr---1 × sizeof(type) bytes移到上一個元素
ptr1 - ptr2兩指標相差幾個元素計算陣列距離

指標與陣列
05

指標與陣列

Pointers & Arrays — 陣列名稱就是指標常數

🎯
核心定理

陣列名稱本身就是指標常數，指向第一個元素。arr[i] 完全等價於 *(arr + i)。這就是為什麼陣列索引從 0 開始！

C++ — 傳統陣列寫法
int arr[] = {1, 2, 3};
for(int i=0; i<3; i++) {
    cout << arr[i];
}
C++ — 指標寫法（等價）
int arr[] = {1, 2, 3};
for(int i=0; i<3; i++) {
    cout << *(arr+i);
}

🔁 用指標遍歷陣列

C++ — 指標遍歷 
#include <iostream>
using namespace std;

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int n = 5;

    // 方法 1：索引法（最常見）
    for (int i = 0; i < n; i++) {
        cout << arr[i] << " ";   // 10 20 30 40 50
    }
    cout << endl;

    // 方法 2：指標法（等價）
    int* ptr = arr;
    for (int i = 0; i < n; i++) {
        cout << *(ptr + i) << " ";  // 相同結果
    }
    cout << endl;

    // 方法 3：指標遞增法
    for (int* p = arr; p < arr + n; p++) {
        cout << *p << " ";  // 最常見的指標遍歷方式
    }

    return 0;
}

06

傳值 vs 傳址

Pass by Value vs Pass by Pointer — 函數參數的本質差異

🔄 差異說明

項目傳值（Value）傳址（Pointer）Python 可變物件
修改是否影響原變數❌ 不影響✅ 會影響✅ 會影響
記憶體行為複製一份資料共用同一塊記憶體共用同一物件
APCS 常考—🔥 swap 函數list/dict 修改副作用
C++ — 傳值 vs 傳址（swap 示範） 
#include <iostream>
using namespace std;

// ❌ 傳值 — 只改複本，無效
void swapBad(int a, int b) {
    int tmp = a;
    a = b;
    b = tmp;
    // 函數結束後 a, b 消滅，外面不變！
}

// ✅ 傳址 — 直接操作原始記憶體
void swapGood(int* a, int* b) {
    int tmp = *a;  // tmp = a 指向的值
    *a = *b;      // 修改 a 指向的記憶體 = b 指向的記憶體
    *b = tmp;     // 修改 b 指向的記憶體
}

int main() {
    int x = 5, y = 15;

    swapBad(x, y);
    cout << "傳值後 x=" << x << " y=" << y << endl; // x=5, y=15（沒變！）

    swapGood(&x, &y);
    cout << "傳址後 x=" << x << " y=" << y << endl; // x=15, y=5（成功！）

    return 0;
}
Python — 可變 vs 不可變物件 
# Python 整數是不可變物件 → 傳入函數不會改變外部
def swap_bad(a, b):
    a, b = b, a          # 只改了本地變數，外部不受影響

x, y = 5, 15
swap_bad(x, y)
print(x, y)              # 5 15（沒變！）

# ✅ Python 的 swap 慣用法
x, y = y, x
print(x, y)              # 15 5（成功！）

# List 是可變物件 → 傳入函數會影響外部（類似傳址）
def modify_list(lst):
    lst[0] = 999         # 直接修改原 list

nums = [1, 2, 3]
modify_list(nums)
print(nums)              # [999, 2, 3]（被改了！）

Linked List — 指標最重要的實際應用

🔗
鏈結串列結構

每個節點（Node）包含兩部分：資料（data） + 下一個節點的位址（next pointer）。最後一個節點的 next 指向 NULL（或 -1）。

Linked List 節點示意
head
data: 'c'
next → 1
→
index 1
data: 'a'
next → 2
→
index 2
data: 'b'
next → -1
→
NULL / -1
結束

🗑️ RemoveNextElement — 刪除節點

要刪除 current 的下一個節點，核心邏輯是：讓 current 直接跳過下一個節點，指向下下個。

刪除前：
c(current) → a（要刪） → b
刪除後：
c(current) → b
只需：current.next = current.next.next
C++ — 陣列模擬 Linked List 
struct element {
    char data;
    int  next;  // 下一個節點的 index，-1 表示結尾
};

void RemoveNextElement(element list[], int current) {
    if (list[current].next != -1) {          // 先確認有下一個節點
        // 核心：讓 current 的 next 跳過下一個節點
        list[current].next = list[list[current].next].next;
        //              ↑現在的next        ↑next的next
    }
}

// 使用示範：
// list[3] = 'c', next=1
// list[1] = 'a', next=2
// list[2] = 'b', next=-1
// 串列：c → a → b
//
// 呼叫 RemoveNextElement(list, 3) → 刪掉 a
// 串列變成：c → b
Python — 指標物件 Linked List 
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

def remove_next(current):
    if current.next is not None:
        current.next = current.next.next  # 直接跳過下一個

# 建立串列：c → a → b
b = Node('b')
a = Node('a'); a.next = b
c = Node('c'); c.next = a

# 印出串列
def print_list(head):
    node = head
    parts = []
    while node:
        parts.append(node.data)
        node = node.next
    print(" → ".join(parts))

print_list(c)          # c → a → b
remove_next(c)         # 刪掉 a
print_list(c)          # c → b

APCS 模擬試題

歷屆真題詳解 — 點選選項後查看解析

觀念題 105年10月 APCS 高頻
右列程式片段中，假設 a, a_ptr 和 a_ptrptr 這三個變數都有被正確宣告，
且呼叫 G() 函式時的參數為 a_ptr 及 a_ptrptr。
G() 函式的兩個參數型態該如何宣告？
題目程式碼
void G((a) a_ptr, (b) a_ptrptr) {
    // ...
}
void main() {
    int a = 1;
    // a_ptr, a_ptrptr 已宣告
    a_ptr = &a;
    a_ptrptr = &a_ptr;
    G(a_ptr, a_ptrptr);
}

✅ 正確答案：(D) (a) int*，(b) int**

1

分析 a_ptr = &a

a 是 int，所以 &a 是 int*（int 的位址）。
因此 a_ptr 的型態是 int* → 函數參數 (a) = int*

2

分析 a_ptrptr = &a_ptr

a_ptr 是 int*，所以 &a_ptr 是 int**（int* 的位址）。
因此 a_ptrptr 的型態是 int** → 函數參數 (b) = int**

3

為何 (A)(B) 錯？

*int 是非法語法！C++ 中 * 必須在型態後面（int*），不能在前面。

🔑 記憶口訣

  • 看到 & 就多一層 *
  • a → int，&a → int*，&(int*) → int**
  • 函數參數型態要和傳入的東西一致
觀念題 Linked List 操作 APCS 高頻
List 是一個陣列，裡面的元素是 element，定義如下。每個 element 利用 next 整數變數記錄下一個 element 在陣列中的位置，-1 表示沒有下一個。

初始資料：index 1 → 'a'(next=2)，index 2 → 'b'(next=-1)，index 3 → 'c'(next=1)
串列：c → a → b

RemoveNextElement 移除 current 所指向的下一個元素，但保持原始串列順序。
若 current=3，呼叫後串列應為 c → b。請問空白處應填入？

✅ 正確答案：(B)

1

找「要刪掉的節點」

list[current].next = current 的下一個節點的 index（也就是要刪的那個）

2

找「刪掉後要接上的節點」

list[list[current].next].next = 要刪節點的下一個（也就是要接上的）

3

讓 current 直接跳過

list[current].next = list[list[current].next].next;
等同於：A.next = B.next（跳過 B 直接接 C）

❗
其他選項錯誤原因
  • (A) 讓 current 指向自己 → 無限迴圈
  • (C) 只改了局部變數 current，沒有改到串列結構
  • (D) 改的對象弄反了，造成錯誤結構

考前必背 · APCS 考試導向速查

🔥
必背觀念（考前 10 分鐘速記）
  • 指標儲存位址，* 用來取值，& 用來取址
  • 未初始化的指標 = 野指標，極度危險（APCS 高頻錯誤）
  • 陣列名稱就是指標常數，arr[i] ≡ *(arr + i)
  • 每多一個 * 就多一層間接參考
  • 指標運算的單位是資料型態的大小（sizeof）
  • 傳址呼叫能讓函數修改原始變數（swap 必用）

操作C++ 語法說明
宣告指標int* p;宣告指向 int 的指標（未初始化！危險）
安全初始化int* p = nullptr;指向空，避免野指標
取址p = #p 儲存 num 的位址
取值*p取出 p 所指向的值

雙層取值**pp兩層間接取值
指標移動p++移到下一個元素（+sizeof(int) bytes）
陣列存取*(arr+i)等同 arr[i]

🚨 陷阱大全

❗
APCS 常見錯誤（每題都可能考）
  • int *x, y; → y 是普通 int，不是指標！要寫 int *x, *y;
  • int *p = 10; → 非法！指標要給位址，不是整數值
  • 野指標（未初始化就取值）→ Segmentation fault 崩潰
  • 對 nullptr 取值 *p → 崩潰
  • *int 是非法語法，正確是 int*
  • 傳值 swap 無效 → 必須用指標（傳址）才能真正交換
